## Methods

### addRenderTree

```typescript
async *addRenderTree(subtreeId: string): AsyncGenerator<any, void, unknown>
```

Generator function which takes the id of a render tree, builds the batches, then adds the batches to the scene. The function will `yield` after each batch has been created, allowing for user defined code to be run in the following fashion.

```typescript
for await (const step of speckleRenderer.addRenderTree(id)) {
  // User defined code
}
```

**Parameters**

- **subtreeId**: The id of the render tree to add to the scene

**Returns**: [**_AsyncGenerator_**](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/AsyncGenerator)

### boxFromObjects

### setMaterial

There are several overloads of this method.

```typescript
setMaterial(rvs: NodeRenderView[], material: Material): void
```

Sets the material instance to the specified rvs.

**Parameters**

- **rvs**: [_NodeRenderView_](/dev/viewer/render-view-api)
- **material**: The material instance to apply. It can be a vanilla three.js [_Material_](https://threejs.org/docs/index.html?q=mate#api/en/materials/Material) but also a [_SpeckleMaterial_](/dev/viewer/speckle-material-api)

```typescript
setMaterial(
    rvs: NodeRenderView[],
    material: RenderMaterial & DisplayStyle & MaterialOptions
): void
```

Creates a material based on the intersection between [_RenderMaterial_](/dev/viewer/speckle-material-api#rendermaterial), [_DisplayStyle_](/dev/viewer/speckle-material-api#displaystyle) and [_MaterialOptions_](/dev/viewer/speckle-material-api#materialoptions). Because this method does not discriminate based on the render view's geometry type (mesh, lines, points) it needs to be able to build materials suitable for all gometry types.

**Parameters**

- **rvs**: [_NodeRenderView_](/dev/viewer/render-view-api)
- **material**: [_RenderMaterial_](/dev/viewer/speckle-material-api#rendermaterial) & [_DisplayStyle_](/dev/viewer/speckle-material-api#displaystyle) & [_MaterialOptions_](/dev/viewer/speckle-material-api#materialoptions)

**Returns**: _void_

```typescript
setMaterial(rvs: NodeRenderView[], material: FilterMaterial): void
```

Sets the filter material to the specified rvs. [_FilterMaterial_](/dev/viewer/speckle-material-api#filterMmaterial)s are a set of predefined material types which are commonly used.

**Parameters**

- **rvs**: [_NodeRenderView_](/dev/viewer/render-view-api)
- **material**: [_FilterMaterial_](/dev/viewer/speckle-material-api#filterMmaterial)

**Returns**: _void_
