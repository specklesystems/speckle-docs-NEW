title: Monitoring
description: Monitor your Speckle server deployment
---

## Health Checks

Check server health:
```bash
curl http://localhost:3000/health
```

## Logs

View application logs:
```bash
# Docker logs
docker-compose logs speckle-server

# Follow logs
docker-compose logs -f speckle-server
```

## Database Monitoring

Check database status:
```bash
# Connect to database
docker-compose exec postgres psql -U speckle -d speckle

# Check active connections
SELECT count(*) FROM pg_stat_activity WHERE state = 'active';
```

## Key Metrics

Monitor these key metrics:
- Server response time
- Database connection count
- Disk usage
- Memory usage
- Error rates

## Alerts

Set up alerts for:
- Server downtime
- High error rates
- Database connection issues
- Disk space warnings
